1.1 PURPOSE

This document defines requirements for Geography planning in Step 4. Geography planning establishes WHERE the campaign will run and provides the foundation for budget allocation and outcome projections by market.

1.2 CORE PRINCIPLE

Geography is not just a targeting input - it is an analytical framework. Every geography decision must be justified with:
- Census-validated population data
- Target audience sizing with calculation methodology
- Projected outcomes
- Structured table format with rollup calculations and source citations

1.3 MANDATORY CENSUS REQUIREMENT

The agent MUST web search for current census data. Estimates without census validation are not acceptable. This is not optional.

Primary: US Census Bureau American Community Survey
- Search: site:census.gov [DMA name] population American Community Survey

- Search: Nielsen DMA [market name] population households

2.2 CANADA

Primary: Statistics Canada
- Search: site:statcan.gc.ca [CMA name] population census profile

2.3 UNITED KINGDOM

Primary: Office for National Statistics
- Search: site:ons.gov.uk [region] population census

2.4 EUROPEAN UNION

Primary: Eurostat
- Search: site:ec.europa.eu/eurostat [country] population

National Offices:
- Germany: site:destatis.de [bundesland] population
- France: site:insee.fr [region] population recensement
- Spain: site:ine.es [comunidad] poblacion

2.5 OTHER REGIONS

Australia: site:abs.gov.au [state] population census
Mexico: site:inegi.org.mx [estado] poblacion censo
Brazil: site:ibge.gov.br [estado] populacao censo
Fallback: World Bank population data [country] [year]

2.6 CITATION FORMAT

Every population figure must include citation with:
- Source name (US Census ACS, Statistics Canada, etc.)
- Geographic level (DMA, State, CMA)
- Data year or range
- Table reference if applicable

3.1 LEVEL 1 - COUNTRY

- Simplest level for national campaigns
- Single row plus total
- Used when no geographic variation in strategy

3.2 LEVEL 2 - DMA

- Designated Market Area level
- Standard for most campaigns
- 210 DMAs in United States
- Rolls up to Country total

3.3 LEVEL 3 - DMA PLUS CORRIDOR

- DMA level with corridor subdivisions
- Corridors are named geographic clusters within DMAs
- Each corridor rolls up to parent DMA
- DMAs roll up to Country total

3.4 LEVEL 4 - ZIP CODES

- Most granular level
- ZIP codes roll up to parent DMA
- Used for hyperlocal campaigns
- Requires first-party data for precision

4.1 PHASE 1 COLUMNS - GEOGRAPHY DEFINITION

Present when Step 4 Geography is being defined:
- Geography Name
- Parent Geography (for rollup)
- Total Population (census data with citation)
- Target Audience Size (after demographic/behavioral filters)
- Target Audience Percent

4.2 PHASE 2 COLUMNS - POST BUDGET ALLOCATION

Add once Step 5 Budget is complete:
- Planned Media Spend
- Spend Per Target
- Projected Impressions
- Projected Conversions
- Projected CAC/CPA

4.3 ROLLUP RULES

SUM aggregation: Population, Audience Size, Spend, Impressions, Conversions
WEIGHTED AVERAGE: Target Percent, Spend Per Target, CAC/CPA

Always include:
- Subtotal row after each DMA grouping (for ZIP/Corridor tables)
- Grand total row at bottom
- Data Sources section with census citations

5.1 CALCULATION METHODOLOGY

Start with total population from census.
Apply each signal as filter with explicit percentage.
Show calculation chain at each step.
Result is target audience size.

5.2 EXAMPLE CALCULATION

Los Angeles DMA Audience Sizing:
- Total population: 13,200,000 (Source: US Census ACS 2019-2023)
- Age 25 to 54: 40% of population = 5,280,000
- HHI over 75K: 50% of age-qualified = 2,640,000
- Hispanic origin: 45% of income-qualified = 1,188,000
- In-market for financial services: 8% = 95,040
- Target audience: 95,040 (0.7% of total DMA population)

6.1 FLAGS TO RAISE

- Total budget in geography table does not equal campaign budget
- Target audience size exceeds total population for any geography
- Spend per target varies more than 50% across geographies
- Population data has no census citation
- Any geography has zero spend but is included in table
