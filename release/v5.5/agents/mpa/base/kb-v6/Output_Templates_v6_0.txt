1.1 PURPOSE

This document defines templates and formatting specifications for user-facing outputs. Unlike knowledge base files which follow 6-Rule Compliance for agent parsing, user-facing outputs use rich formatting for professional presentation.

1.2 DOCUMENT CATEGORIES

Media Plan (.docx) - Gate 4 completion - 15-25 pages
Executive Summary (.docx) - On demand - 2-3 pages
In-Flight Optimization Report (.docx) - Weekly - 8-12 pages
Post-Mortem Report (.docx) - Campaign completion - 12-20 pages
Budget Allocation Summary (.docx/.xlsx) - Steps 3/5 - 2-4 pages
Performance Dashboard (.pptx) - Weekly - 5-10 slides

1.3 FORMATTING PRINCIPLES

CLARITY FIRST:
- Information hierarchy immediately apparent
- Key data points easily scannable
- Dense content broken into digestible sections

PROFESSIONAL APPEARANCE:
- Consistent formatting throughout
- Clean, modern aesthetic
- Appropriate use of white space

ACCESSIBILITY:
- Color contrast meets WCAG 2.1 AA standards
- Minimum 10pt for body text
- Alternative text for visualizations

6.1 STANDARD DATA TABLE

Header Row: Bold, Primary Blue background (#1E3A5F), white text
Data Rows: Alternating white and light gray (#F7FAFC)
Borders: Light gray (#E2E8F0), 0.5pt
Column Widths: Auto-fit or proportional to content
Alignment: Text left, numbers right, currency right

6.2 KPI SUMMARY TABLE

Metric Name | Target | Actual | Variance | Status
Left align | Right | Right | Right | Center (icon)
Status icons: Green check, yellow warning, red X

6.3 BUDGET ALLOCATION TABLE

Channel | Budget | Percent | CPM | Impressions | Conversions
Include totals row with bold formatting
Percentage column should sum to 100%
