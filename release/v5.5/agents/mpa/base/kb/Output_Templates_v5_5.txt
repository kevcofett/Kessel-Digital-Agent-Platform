## SECTION 1: DOCUMENT OVERVIEW

### 1.1 Purpose

This document defines the complete formatting specifications and templates for all user-facing outputs generated by the Media Planning Agent. Unlike knowledge base files which follow the 6-Rule Compliance format for agent parsing, user-facing outputs utilize full Microsoft rich text formatting to deliver professional, branded, and visually compelling documents.

### 1.2 Document Categories

The Media Planning Agent generates the following document types:

| Document Type | Primary Format | When Generated | Typical Length |
|--------------|----------------|----------------|----------------|
| Media Plan | .docx | Gate 4 completion | 15-25 pages |
| Executive Summary | .docx | On demand | 2-3 pages |
| In-Flight Optimization Report | .docx | Weekly/On demand | 8-12 pages |
| Post-Mortem Report | .docx | Campaign completion | 12-20 pages |
| Channel Playbook Export | .docx | On demand | 5-8 pages |
| Budget Allocation Summary | .docx/.xlsx | Steps 3 and 5 | 2-4 pages |
| Performance Dashboard | .pptx | Weekly/On demand | 5-10 slides |

### 1.3 Formatting Philosophy

All user-facing documents adhere to these principles:

**CLARITY FIRST**
- Information hierarchy is immediately apparent
- Key data points are easily scannable
- Dense content is broken into digestible sections

**PROFESSIONAL APPEARANCE**
- Consistent formatting throughout
- Clean, modern aesthetic
- Appropriate use of white space

**BRAND FLEXIBILITY**
- Default styling works for all clients
- Easy customization for branded outputs
- Client colors and logos integrate seamlessly

**ACCESSIBILITY**
- Color contrast meets WCAG 2.1 AA standards
- Readable font sizes (minimum 10pt for body)
- Alternative text for all visualizations

## SECTION 2: TYPOGRAPHY STANDARDS

### 2.1 Primary Font Family

**Primary Font:** Arial (cross-platform compatibility)  
**Fallback Fonts:** Helvetica, Calibri, sans-serif

### 2.2 Heading Hierarchy

| Element | Font | Size | Weight | Color | Spacing After |
|---------|------|------|--------|-------|---------------|
| Document Title | Arial | 24pt | Bold | Primary Blue (#1E3A5F) | 18pt |
| Heading 1 | Arial | 18pt | Bold | Primary Blue (#1E3A5F) | 12pt |
| Heading 2 | Arial | 14pt | Bold | Secondary Blue (#2C5282) | 10pt |
| Heading 3 | Arial | 12pt | Bold | Dark Gray (#2D3748) | 8pt |
| Heading 4 | Arial | 11pt | Bold | Dark Gray (#2D3748) | 6pt |
| Body Text | Arial | 11pt | Regular | Black (#1A202C) | 6pt |
| Body Text (Dense) | Arial | 10pt | Regular | Black (#1A202C) | 4pt |
| Caption | Arial | 9pt | Italic | Medium Gray (#718096) | 4pt |
| Footnote | Arial | 8pt | Regular | Medium Gray (#718096) | 2pt |

### 2.3 Paragraph Formatting

**Body Paragraphs:**
- Line spacing: 1.15
- Space after paragraph: 6pt
- First line indent: None
- Alignment: Left (justified for formal documents)

**Bulleted Lists:**
- Bullet character: Solid circle ([STATUS])
- Indent: 0.25 inches
- Hanging indent: 0.25 inches
- Space after item: 3pt

**Numbered Lists:**
- Format: 1. 2. 3. (or a. b. c. for sub-lists)
- Indent: 0.25 inches
- Hanging indent: 0.25 inches
- Space after item: 3pt

### 2.4 Special Text Treatments

**Key Metrics:**
- Font: Arial Bold
- Size: 14pt (inline) or 24pt (callout boxes)
- Color: Primary Blue or Accent color based on context

**Data Labels:**
- Font: Arial
- Size: 10pt
- Weight: Regular or Bold depending on importance

**Hyperlinks:**
- Color: Link Blue (#3182CE)
- Underline: Yes
- Hover state: Darker blue (#2B6CB0)

**Emphasis:**
- **Bold:** For key terms, important values, column headers
- *Italic:* For definitions, technical terms, citations
- Underline: Reserved for hyperlinks only

### 3.2 Brand Integration

When client branding is provided, the following elements are customized:

**Cover Page:**
- Client logo: Top left or centered, maximum height 1.5 inches
- Primary brand color: Applied to title and main header
- Secondary brand color: Applied to accent elements

**Headers and Footers:**
- Header: Document title (left), Client logo (right) if provided
- Footer: Page number (center), Date (right)

**Color Application Rules:**
- If client provides only one brand color: Use for primary elements, maintain default blues for secondary
- If client provides two colors: Map to Primary and Accent positions
- If client provides full palette: Apply per client style guide
- Always maintain sufficient contrast for accessibility (minimum 4.5:1 ratio)

### 3.3 Chart Color Sequences

**Multi-Series Charts (up to 8 series):**
- #3182CE (Blue)
- #38A169 (Green)
- #D69E2E (Yellow)
- #E53E3E (Red)
- #805AD5 (Purple)
- #DD6B20 (Orange)
- #00B5D8 (Cyan)
- #ED64A6 (Pink)

**Performance Comparison (Planned vs Actual):**
- Planned: #3182CE (Solid blue)
- Actual: #38A169 (Solid green) if meeting target, #E53E3E (Solid red) if below

**Budget Allocation:**
- Use sequential blues for channel breakdown
- Darkest to lightest based on allocation percentage

## SECTION 5: DATA VISUALIZATION STANDARDS

### 5.1 General Chart Guidelines

**All Charts Must Include:**
- Clear, descriptive title (Heading 3 style)
- Axis labels with units
- Legend (when multiple series)
- Data labels for key values
- Source citation if external data

**Chart Sizing:**
- Full width: 6.5 inches (standard page width)
- Half width: 3.0 inches (for side-by-side)
- Height: Proportional, typically 3-4 inches
- Minimum font size in charts: 9pt

**Accessibility Requirements:**
- Never rely solely on color to convey information
- Use patterns or labels in addition to colors
- Maintain sufficient color contrast

### 5.2 Bar Charts

**Use Case:** Comparing values across categories (channels, audiences, time periods)

**Specifications:**

| Element | Specification |
|---------|--------------|
| Orientation | Horizontal preferred for category labels, Vertical for time series |
| Bar Width | 0.5 inches minimum |
| Gap Between Bars | 0.1 inches |
| Gap Between Groups | 0.2 inches (grouped bars) |
| Data Labels | Positioned at end of bar or inside if sufficient width |
| Grid Lines | Horizontal only, Light Gray (#E2E8F0), dashed |
| Baseline | Solid black line at zero |

**Color Application:**
- Single series: Primary Blue (#3182CE)
- Multiple series: Use chart color sequence
- Highlighting: Use Accent color for featured bar

**Example Use Cases:**
- Channel budget allocation comparison
- KPI performance by channel
- Audience segment size comparison
- Partner performance ranking

### 5.3 Line Charts

**Use Case:** Showing trends over time

**Specifications:**

| Element | Specification |
|---------|--------------|
| Line Width | 2pt primary, 1.5pt secondary |
| Line Style | Solid for actual, Dashed for projected/target |
| Data Points | Shown as circles (6pt diameter) on key points |
| Grid Lines | Both axes, Light Gray (#E2E8F0), dashed |
| Axis Format | Time on X-axis, Metric on Y-axis |

**Color Application:**
- Actual performance: Primary Blue (#3182CE)
- Target/Planned: Medium Gray (#718096) dashed
- Projected: Accent Blue (#3182CE) dashed with lighter opacity
- Multiple metrics: Use chart color sequence

**Example Use Cases:**
- Campaign performance over time
- Spend pacing visualization
- CTR/CVR trend analysis
- Cumulative metrics tracking

### 5.4 Pie/Donut Charts

**Use Case:** Showing composition or allocation percentages

**Specifications:**

| Element | Specification |
|---------|--------------|
| Chart Type | Donut preferred (cleaner appearance) |
| Donut Hole Size | 50% of diameter |
| Segment Minimum | 5% (smaller segments grouped as "Other") |
| Labels | Percentage and category name |
| Legend | Positioned right or below |
| Maximum Segments | 6-8 (beyond this, use bar chart) |

**Color Application:**
- Use chart color sequence
- Start with Primary Blue, progress through palette
- "Other" category: Light Gray (#E2E8F0)

**Example Use Cases:**
- Budget allocation by channel
- Traffic source breakdown
- Audience segment composition
- Device type distribution

### 5.5 Stacked Bar/Area Charts

**Use Case:** Showing composition over time or across categories

**Specifications:**

| Element | Specification |
|---------|--------------|
| Stacking | 100% stacked for proportions, Absolute for totals |
| Segment Order | Largest on bottom, smallest on top |
| Labels | Show percentage or value if space permits |
| Totals | Display above each bar/column |

**Color Application:**
- Use chart color sequence from bottom to top
- Maintain consistent colors across time periods

**Example Use Cases:**
- Channel mix evolution over campaign
- Funnel stage breakdown
- Spend allocation by week

### 5.6 Gauge/Progress Charts

**Use Case:** Showing progress toward a single target

**Specifications:**

| Element | Specification |
|---------|--------------|
| Chart Type | Semi-circle gauge or linear progress bar |
| Target Line | Solid black vertical line at target position |
| Zones | Green (above target), Yellow (within 10%), Red (below 90%) |
| Current Value | Large, bold number in center or beside |
| Percentage | Shown as X% of target |

**Example Use Cases:**
- Budget pacing
- KPI attainment
- Campaign completion status

### 5.7 Scatter/Bubble Charts

**Use Case:** Showing relationships between two or three variables

**Specifications:**

| Element | Specification |
|---------|--------------|
| Axis Labels | Clear variable names with units |
| Data Points | Minimum 8pt diameter |
| Bubble Size | Third variable, with legend showing scale |
| Quadrant Lines | If applicable, show target thresholds |
| Labels | Show for key data points only |

**Example Use Cases:**
- Efficiency matrix (CPM vs CTR by channel)
- Investment vs Return analysis
- Audience size vs conversion rate

### 5.8 Funnel Charts

**Use Case:** Showing conversion through stages

**Specifications:**

| Element | Specification |
|---------|--------------|
| Orientation | Vertical (top to bottom) |
| Stage Labels | Left of funnel |
| Values | Right of funnel (absolute and percentage) |
| Drop-off | Show percentage lost between stages |
| Width Proportion | Proportional to stage volume |

**Color Application:**
- Gradient from Primary Blue (top) to Accent Blue (bottom)
- Or single color with decreasing opacity

**Example Use Cases:**
- Marketing funnel visualization
- Conversion path analysis
- Lead qualification stages

### 5.9 Heat Maps

**Use Case:** Showing performance across two dimensions

**Specifications:**

| Element | Specification |
|---------|--------------|
| Cell Colors | Gradient from low (red/white) to high (green/blue) |
| Cell Values | Numeric value in each cell |
| Row/Column Headers | Clear labels |
| Legend | Color scale with min/max values |

**Example Use Cases:**
- Day-of-week and hour performance
- Channel by audience performance
- Geographic performance comparison

### 5.10 Sparklines

**Use Case:** Compact trend visualization within tables

**Specifications:**

| Element | Specification |
|---------|--------------|
| Size | Inline with table cell, approximately 0.25 inches height |
| Color | Single color matching context |
| Data Points | First, last, min, max may be highlighted |
| No Axes | Clean line only |

**Example Use Cases:**
- Trend column in KPI tables
- Quick performance indicators
- Historical context in summary tables

Instead of: "Foundation phase | EXPRESS | Projected: 598 leads @ $418 CAC"

- Foundation phase: Steps 1-3 (Context, Objectives, Budget)
- Strategy phase: Steps 4-6 (Audience, Channels, Partners)
- Execution phase: Steps 7-9 (Measurement, Optimization, Compliance)
- Finalization: Step 10 (Document generation)

NATURAL PHRASES BY PHASE:

FOUNDATION PHASE:
- "Capturing the basics..."
- "Foundation set. Moving to strategy."
- "Got the essentials. Ready to build the plan."

STRATEGY PHASE:
- "Audience and channels locked."
- "Strategy taking shape."
- "Building out the allocation now."

EXECUTION PHASE:
- "Rounding out the details."
- "Measurement approach set."
- "Almost there - final checks."

FINALIZATION:
- "Complete plan ready for review."
- "Ready to generate your media plan document."
- "All set. Approve to finalize?"

SOFT CHECKPOINT LANGUAGE:

Instead of: "GATE 1 - FOUNDATION VALIDATION"

Use: "Before we dive into channels, let me confirm I have the foundation right..."

Instead of: "Do you approve Gate 1 Foundation to proceed?"

Use: "Does this capture your situation? I'll build the strategy from here."

FORMAL GATE (only for final document):

- "Plan ready for export. Final approval needed before I generate the document."
- "Ready to create the media plan document. Approve to proceed?"
