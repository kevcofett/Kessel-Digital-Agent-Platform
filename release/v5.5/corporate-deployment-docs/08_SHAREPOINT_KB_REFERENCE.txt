LIBRARY CONFIGURATION
============================================================

CREATE DOCUMENT LIBRARY

Navigation
1 Open SharePoint site
2 Click Settings gear
3 Click Site contents
4 Click New
5 Click Document library

Library Settings
- Name - MediaPlanningKB
- Description - Knowledge base for Media Planning Agent v5.5
- Navigation - Show in site navigation (optional)
- Versioning - Enable versioning (recommended)

PERMISSIONS

Required Permissions
- Copilot service account needs Read access
- Administrators need Full Control
- Content editors need Contribute

Configure Permissions
1 Open library settings
2 Click Library permissions
3 Click Stop Inheriting Permissions (if needed)
4 Add required users and groups
5 Set appropriate permission levels

============================================================
FILE UPLOAD PROCESS
============================================================

MANUAL UPLOAD

Steps
1 Open SharePoint site
2 Navigate to MediaPlanningKB library
3 Click Upload
4 Click Files
5 Select all 22 TXT files from kb folder
6 Click Open
7 Wait for upload to complete
8 Verify all files appear in library

Source Location
release/v5.5/agents/mpa/base/kb/

SCRIPT UPLOAD

Prerequisites
- Python 3.9 or higher
- Microsoft Graph SDK installed
- Network access to SharePoint

Script Location
release/v5.5/scripts/upload_kb_files.py

Usage
python upload_kb_files.py
python upload_kb_files.py --overwrite

Authentication
- Script uses device code flow
- User authenticates via browser
- Requires SharePoint write permissions

============================================================
INDEXING
============================================================

INDEXING TIMELINE

Initial Indexing
- Starts automatically after upload
- Takes 1 to 4 hours to complete
- Larger libraries take longer

Incremental Updates
- New or modified files indexed within 24 hours
- Manual reindex available if needed

VERIFY INDEXING

In Copilot Studio
1 Open Media Planning Agent
2 Click Knowledge
3 Check knowledge source status
4 Verify indexing percentage

Test Search
1 Open agent test panel
2 Ask question about specific KB topic
3 Verify agent references KB content
4 Check source citation appears

FORCE REINDEX

If indexing stuck
1 Open Copilot Studio
2 Click Knowledge
3 Click on SharePoint source
4 Click Refresh or Reindex option
5 Wait for reindexing to complete

============================================================
CONTENT GUIDELINES
============================================================

FILE FORMAT REQUIREMENTS

Format
- Plain text only (TXT extension)
- UTF-8 encoding
- No special characters
- No formatting markup

Structure
- Clear section headers
- Consistent formatting
- Searchable keywords
- Complete sentences

Content Quality
- Accurate information
- Current data
- Professional tone
- Actionable guidance

SIZE LIMITS

Per File
- Maximum 5 MB per file
- Recommended under 100 KB

Total Library
- No hard limit for library size
- Performance may degrade with very large libraries

NAMING CONVENTIONS

File Names
- Use underscores not spaces
- Include version number
- Use lowercase or mixed case
- Include TXT extension

Examples
- MPA_Expert_Lens_Budget_Allocation_v5_5.txt
- Strategic_Wisdom_v5_5.txt
- MEASUREMENT_FRAMEWORK_v5_5.txt

CONTENT REFRESH

============================================================
TROUBLESHOOTING
============================================================

FILES NOT APPEARING IN SEARCH

Check
- File format is TXT
- File is not empty
- Indexing has completed
- No special characters in content

Resolution
- Convert to plain text if needed
- Allow time for indexing
- Force reindex if stuck

WRONG CONTENT RETURNED

Check
- File content is accurate
- No conflicting information
- Keywords are appropriate

Resolution
- Review and update content
- Add clarifying context
- Improve keyword coverage

INDEXING FAILURES

Check
- SharePoint permissions
- File accessibility
- Library configuration

Resolution
- Verify service account access
- Check library settings
- Contact SharePoint admin
