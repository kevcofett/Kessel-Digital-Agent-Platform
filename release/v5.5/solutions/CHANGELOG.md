## Version Numbering

Solutions use four-part version numbers: `Major.Minor.Build.Revision`

| Part | When to Increment |
|------|------------------|
| Major | Breaking changes, major restructuring |
| Minor | New features, significant changes |
| Build | Bug fixes, minor improvements |
| Revision | Internal builds, hotfixes |

**Examples:**
- `1.0.0.0` → Initial release
- `1.1.0.0` → Added new topic
- `1.1.1.0` → Fixed trigger phrase
- `1.1.1.1` → Quick hotfix
