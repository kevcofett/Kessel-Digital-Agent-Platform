SECTION 1 - DOCUMENT FORMATTING STANDARDS
================================================================================

TYPOGRAPHY STANDARDS

FONT HIERARCHY

DOCUMENT TITLE
- Font: Arial Bold or Calibri Bold
- Size: 24-28 point
- Color: Primary brand color or black
- Alignment: Left or center

SECTION HEADERS (H1)
- Font: Arial Bold or Calibri Bold
- Size: 16-18 point
- Color: Primary brand color or black
- Spacing: 18pt before, 6pt after

SUBSECTION HEADERS (H2)
- Font: Arial Bold or Calibri Bold
- Size: 14 point
- Color: Dark gray or black
- Spacing: 12pt before, 6pt after

BODY TEXT
- Font: Arial or Calibri
- Size: 11 point
- Color: Black
- Line spacing: 1.15 or 1.5
- Paragraph spacing: 6pt after

TABLE TEXT
- Font: Arial or Calibri
- Size: 10 point
- Color: Black
- Row height: Auto or 0.3 inches minimum

CAPTION TEXT
- Font: Arial Italic or Calibri Italic
- Size: 9 point
- Color: Dark gray

PAGE LAYOUT

PAGE SIZE
- Standard: Letter (8.5 x 11 inches)
- International: A4 (210 x 297 mm)

MARGINS
- Top: 1 inch (2.54 cm)
- Bottom: 1 inch (2.54 cm)
- Left: 1 inch (2.54 cm)
- Right: 1 inch (2.54 cm)

HEADERS AND FOOTERS
Header content:
- Left: Document title or client name
- Right: Page number of total (Page X of Y)

Footer content:
- Left: Confidentiality notice
- Center: Date
- Right: Version number

SECTION 2 - TABLE FORMATTING
================================================================================

TABLE STRUCTURE RULES

HEADER ROW
- Background: Primary brand color or dark gray
- Text color: White
- Font weight: Bold
- Alignment: Left for text, center for numbers

DATA ROWS
- Background: Alternating white and light gray
- Text color: Black
- Font weight: Regular
- Alignment: Left for text, right for numbers, center for status

TOTAL ROW
- Background: Medium gray
- Text color: Black
- Font weight: Bold
- Border: Top border double line

COLUMN WIDTH GUIDELINES

TEXT COLUMNS
- Channel names: 1.5-2 inches
- Description: 2-3 inches
- Status: 1 inch

NUMERIC COLUMNS
- Currency: 1-1.25 inches
- Percentages: 0.75-1 inch
- Large numbers: 1.25-1.5 inches

NUMBER FORMATTING

CURRENCY
- Format: $X,XXX,XXX
- Decimals: None for amounts over $1,000
- Decimals: Two places for amounts under $1,000
- Negative: Parentheses or red color

PERCENTAGES
- Format: XX.X%
- Decimals: One place standard
- Decimals: None for whole numbers

LARGE NUMBERS
- Format: X,XXX,XXX
- Use K notation for millions (1.5M)
- Use B notation for billions (2.3B)

DECIMALS
- Standard: Two decimal places
- Performance metrics: One decimal place
- Index values: Zero decimal places

SECTION 3 - CHART AND VISUAL FORMATTING
================================================================================

CHART TYPES BY USE CASE

PIE CHARTS
Use for: Budget allocation, channel mix
Maximum slices: 6-7
Below threshold: Group as Other
Label format: Category and percentage

BAR CHARTS
Use for: Comparison across categories
Orientation: Horizontal for many categories, vertical for few
Spacing: 50 percent gap width
Data labels: Include values on bars

LINE CHARTS
Use for: Trends over time
Maximum lines: 4-5
Legend: Below chart
Axis labels: Clear time periods

STACKED BAR CHARTS
Use for: Part-to-whole over categories
Maximum segments: 5
Color coding: Consistent across charts
Totals: Show total label

CHART FORMATTING RULES

COLOR PALETTE
- Primary channel: Brand primary color
- Secondary channels: Brand secondary colors
- Accent: Complementary colors
- Avoid: Red and green together (accessibility)

AXIS FORMATTING
- Y-axis: Start at zero for bar charts
- X-axis: Clear category labels
- Grid lines: Light gray, minimal
- Axis titles: Clear and descriptive

LABELS AND LEGENDS
- Data labels: Show on chart when space allows
- Legend: Position consistently (bottom or right)
- Title: Descriptive, not just Untitled Chart

SECTION 4 - CONTENT WRITING STANDARDS
================================================================================

WRITING STYLE RULES

VOICE AND TONE
- Professional but accessible
- Active voice preferred
- Direct and concise
- Confident but not arrogant

SENTENCE STRUCTURE
- Average sentence length: 15-20 words
- Vary sentence length for rhythm
- Lead with key information
- One idea per sentence

PARAGRAPH STRUCTURE
- Topic sentence first
- Supporting details follow
- Concluding or transition sentence
- 3-5 sentences per paragraph typical

TERMINOLOGY STANDARDS

CHANNEL NAMES
- Use full names on first reference
- Abbreviations acceptable after introduction
- Consistent capitalization throughout

METRIC NAMES
- Spell out on first use with abbreviation
- Example: Cost Per Acquisition (CPA)
- Use abbreviation thereafter

NUMBERS IN TEXT
- Spell out one through nine
- Use numerals for 10 and above
- Always use numerals with units ($500, 25 percent)
- Use numerals in tables

PROJECTION LANGUAGE

CONFIDENCE QUALIFIERS
HIGH confidence: will, expect, project
MEDIUM confidence: likely, should, anticipate
LOW confidence: may, could, potential

RANGE EXPRESSIONS
- Preferred: 1,000-1,200 conversions
- Alternative: 1,000 to 1,200 conversions
- Avoid: approximately 1,100 conversions (too vague)

DISCLAIMER LANGUAGE
Standard disclaimer template:
"Projections are based on [source] benchmarks and [assumptions]. Actual results may vary based on [factors]. We recommend [validation approach]."

SECTION 5 - ACCESSIBILITY STANDARDS
================================================================================

COLOR ACCESSIBILITY

CONTRAST REQUIREMENTS
- Text on background: 4.5:1 ratio minimum
- Large text: 3:1 ratio minimum
- Charts: Distinguishable without color alone

COLOR BLINDNESS CONSIDERATIONS
- Avoid red-green only differentiation
- Use patterns in addition to color
- Test with color blindness simulators

DOCUMENT ACCESSIBILITY

HEADING STRUCTURE
- Use proper heading hierarchy (H1, H2, H3)
- Do not skip heading levels
- Headings describe section content

ALT TEXT FOR IMAGES
- Describe chart content and key takeaway
- Include data summary for complex charts
- Keep under 125 characters when possible

TABLE ACCESSIBILITY
- Define header row
- Keep tables simple (avoid merged cells)
- Include table summary if complex

READABILITY

READING LEVEL
- Target: 8th-10th grade reading level
- Avoid jargon without definition
- Use shorter sentences for complex topics

DOCUMENT STRUCTURE
- Clear section breaks
- Logical flow of information
- Table of contents for long documents

SECTION 6 - VERSION CONTROL AND NAMING
================================================================================

FILE NAMING CONVENTIONS

STANDARD FORMAT
[Client]_[DocumentType]_[CampaignName]_[Version]_[Date]

EXAMPLES
- AcmeCorp_MediaPlan_Q1Launch_v1.0_20260115.docx
- GlobalRetail_ExecSummary_HolidayCampaign_v2.1_20260115.pdf
- TechStart_ChannelBrief_Display_v1.0_20260115.docx

VERSION NUMBERING
- Major version: Significant content changes (v1.0 to v2.0)
- Minor version: Small edits and corrections (v1.0 to v1.1)
- Draft versions: Add _DRAFT suffix

DOCUMENT HISTORY

CHANGE LOG FORMAT
Include in document footer or appendix:
- Version number
- Date
- Author
- Summary of changes

APPROVAL TRACKING
- Draft status until approved
- Approved by and date recorded
- Final version clearly marked
