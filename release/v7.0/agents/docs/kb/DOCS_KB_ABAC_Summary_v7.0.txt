WHAT IS ABAC

MCMAP implements Attribute-Based Access Control integrated with the Mastercard Microsoft Directory. This system protects sensitive content based on user attributes like employee level department division and region.


TWO-MODE OPERATION

LAUNCH MODE (DEFAULT)
- ABAC_ENABLED is false
- Only C-Suite executive documents are protected
- Protected content includes CEO Brief and C-Suite briefs 00-A through 00-D and Investment Proposal

FULL ABAC MODE
- ABAC_ENABLED is true
- All department division and region rules become active
- Additional content protection based on organizational attributes


PROTECTED CONTENT IN LAUNCH MODE

CEO Brief requires Executive access with employee level CEO President EVP or SVP

Chief Consulting Officer Brief requires Executive access

Chief Technology Officer Brief requires Executive access

Chief AI Officer Brief requires Executive access

Chief Revenue Officer Brief requires Executive access

Investment Proposal requires Executive or Finance access


USER ATTRIBUTES CAPTURED

employee_level such as CEO EVP SVP VP Director Manager Analyst

department such as Marketing Services Finance Engineering

division such as Data and Services Network Services

region such as North America Europe APMEA

security_groups from Azure AD for group-based access


ACCESS DENIAL HANDLING

When access is denied

1 Display graceful message saying This content requires specific access permissions

2 Offer access request by saying To request access say request access

3 NEVER reveal Platform Owner email address

4 NEVER disclose specific access conditions

5 NEVER suggest workarounds to bypass controls


ACCESS REQUEST PROCESS

When user says request access

1 Ask what content they want access to

2 Ask for brief justification in 1-2 sentences

3 Submit request via MCMAP_Access_Request flow

4 Confirm with Request ID

5 Mention 2 business day response time


FLOWS INVOLVED

MCMAP_User_Profile_Sync syncs user profile from Microsoft Directory on session start

MCMAP_Check_Content_Access evaluates ABAC rules against user attributes before returning content

MCMAP_Access_Request processes access requests and sends email to Platform Owner


DATAVERSE TABLES

eap_security_config stores global toggles like ABAC_ENABLED

eap_user_profile stores user attributes from directory

eap_access_rule stores ABAC rule definitions

eap_access_request tracks access requests
