EXECUTION MODE PREFERENCES
===========================

Default Mode:
- Quick Mode: Minimal explanations, smart defaults, streamlined
- Full Mode: Detailed explanations, all steps visible (DEFAULT)
- Expert Mode: No explanations unless requested, maximum control

Set Command: set default mode quick|full|expert
View Command: show preferences

Quick Mode Preferences:
- Auto-apply smart defaults: ON|OFF
- Skip Step 6-8 confirmations: ON|OFF
- Framework auto-select threshold: 90pct|95pct
- Minimal explanations: ON|OFF

Full Mode Preferences:
- Show all validation checkpoints: ON|OFF
- Detailed step explanations: ON|OFF
- Framework recommendations: Always|When uncertain
- Progress indicators: ON|OFF

Expert Mode Preferences:
- Direct framework access: ON|OFF
- Override all defaults: ON|OFF
- Explanation triggers: Never|On request|Errors only
- Advanced commands enabled: ON|OFF

PRESET WORKFLOWS
=================

Create Custom Presets:
Save combination of preferences for specific scenarios

Examples:
- Quick Analysis: Quick mode, Concise output, Problem-Solving pack, Minimal explanation
- Strategic Planning: Full mode, Comprehensive output, Strategic pack, Detailed explanation
- Executive Briefing: Expert mode, Balanced output, Strategic plus Customer packs, Executive tone

Create Preset:
save preset name
- Mode: quick|full|expert
- Format: markdown|docx|pptx|json|text
- Packs: List of pack numbers
- Tone: professional|technical|executive
- Optimization: comprehensive|balanced|concise

Load Preset:
load preset name

Manage Presets:
- show presets: List all saved presets
- delete preset name: Remove preset
- update preset name: Modify existing preset

PREFERENCE PERSISTENCE
======================

Preferences Save:
- Automatically saved after each change
- Persist across sessions
- Associated with user role
- Can be exported imported

Preference Scope:
- User-level: Personal preferences
- Role-level: Defaults for role
- Session-level: Temporary for current session only

Preference Priority:
1. Session-level (temporary overrides)
2. User-level (personal preferences)
3. Role-level (role defaults)
4. System-level (global defaults)

BEST PRACTICES
==============

Getting Started:
1. Use role defaults initially
2. Adjust as you learn preferences
3. Create presets for common scenarios
4. Experiment with different settings

For Efficiency:
1. Set default mode to Quick once comfortable
2. Enable auto-load for frequently used modules
3. Create presets for repetitive tasks
4. Use framework pack defaults

For Quality:
1. Keep validation on Auto
2. Enable all quality checks
3. Include executive summaries always
4. Document confidence levels

For Customization:
1. Match technical level to audience
2. Choose appropriate tone
3. Select optimal output format
4. Configure communication depth

TROUBLESHOOTING
===============

Preferences Not Saving:
- Check: User permissions sufficient
- Verify: Preferences command syntax correct
- Try: Reset then reapply preferences

Presets Not Loading:
- Check: Preset name spelled correctly
- Verify: Preset exists (show presets)
- Try: Recreate preset if corrupted

Defaults Not Applying:
- Check: Session overrides active
- Verify: Role permissions allow settings
- Try: Reset preferences to clear conflicts
