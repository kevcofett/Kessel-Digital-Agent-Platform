ORC KNOWLEDGE BASE - ROUTING LOGIC AND INTENT CLASSIFICATION

OVERVIEW

This knowledge base provides detailed routing logic for the Orchestrator Agent to classify user intent and route requests to the appropriate specialist agent. Accurate routing ensures users receive expert assistance while maintaining seamless conversation flow.

MPA DOMAIN ROUTING

The Media Planning Agent domain encompasses analytics, audience, channel, supply path, document, performance, and marketing strategy specialists. These agents support the complete media planning workflow from strategy through execution and measurement.

ANL AGENT TRIGGERS

Route to Analytics Agent when:
- User asks about budget projections or forecasting
- User requests scenario analysis or what-if modeling
- User needs ROI, ROAS, CAC, or LTV calculations
- User asks about statistical significance or incrementality
- User wants diminishing returns or saturation curve analysis
- User requests NPV, IRR, or payback period calculations
- User needs sensitivity analysis or Monte Carlo simulation
- User asks about financial modeling or investment analysis

Keywords: projection, forecast, scenario, model, calculate, ROI, ROAS, CAC, LTV, NPV, IRR, statistical, incrementality, lift, saturation, diminishing returns, sensitivity, financial

AUD AGENT TRIGGERS

Route to Audience Agent when:
- User asks about audience segments or segmentation strategy
- User needs targeting recommendations or audience sizing
- User requests customer value analysis or LTV tiers
- User asks about personas or customer profiles
- User needs lookalike audience guidance or seed quality
- User asks about first-party data activation or clean rooms
- User requests propensity scoring or behavioral analysis

Keywords: audience, segment, targeting, persona, LTV, customer value, lookalike, first-party, clean room, propensity, behavioral, demographics, psychographics

CHA AGENT TRIGGERS

Route to Channel Agent when:
- User asks about channel selection or media mix
- User needs budget allocation across channels
- User requests platform recommendations
- User asks about CPM, CPC, or CPA benchmarks
- User needs frequency or reach guidance
- User asks about channel-specific tactics

Keywords: channel, media mix, allocation, platform, CPM, CPC, CPA, benchmark, frequency, reach, Meta, Google, TikTok, Amazon, programmatic, social, search, display

SPO AGENT TRIGGERS

Route to Supply Path Agent when:
- User asks about programmatic optimization
- User needs fee analysis or transparency assessment
- User requests DSP or SSP evaluation
- User asks about working media ratio
- User needs partner selection guidance
- User asks about PMP versus open exchange

Keywords: supply path, programmatic, DSP, SSP, fees, transparency, working media, tech tax, PMP, exchange, partner, verification

DOC AGENT TRIGGERS

Route to Document Agent when:
- User asks to generate a plan or document
- User requests export or download functionality
- User needs presentation or report creation

- User requests business case or roadmap document

Keywords: generate, create, document, export, download, PDF, PowerPoint, Word, presentation, report, summary, brief, business case, roadmap

PRF AGENT TRIGGERS

Route to Performance Agent when:

MULTI-AGENT SCENARIOS

Some user requests require coordination across multiple agents. The Orchestrator should recognize these patterns and sequence the routing appropriately.

MPA Multi-Agent Patterns:
- Budget optimization: ANL for projections then CHA for allocation
- Audience strategy: AUD for segments then CHA for targeting tactics
- Campaign planning: MKT for strategy then CHA for channels then DOC for document
- Performance review: PRF for analysis then ANL for projections
- Full campaign development: MKT then AUD then CHA then ANL then DOC

MKT Multi-Agent Patterns:
- Campaign strategy with audience: MKT for strategy then AUD for segmentation
- Campaign strategy with channels: MKT for strategy then CHA for channel mix
- Full marketing plan: MKT for strategy then AUD plus CHA plus ANL then DOC

CA Multi-Agent Patterns:
- Business case: CST for framework then ANL for financials then DOC for document
- Transformation: CST for assessment then CHG for change plan then DOC for roadmap
- Strategy implementation: CST for prioritization then CHG for adoption then DOC for brief

Cross-Domain Patterns:
- Marketing transformation: CST plus CHG plus MKT plus MPA agents
- Campaign strategy: MKT for marketing strategy plus AUD plus CHA for execution
- Measurement strategy: CST for approach plus ANL for methodology
- Product launch: MKT for GTM plus CST for strategic framework plus CHG for adoption

ROUTING DECISION LOGIC

Step 1 - Domain Classification
- Media planning, campaigns, advertising - MPA domain
- Marketing strategy, creative briefs, positioning - MKT agent
- Consulting, strategy frameworks, change - CA domain
- General questions, workflow - Handle as ORC

Step 2 - Agent Selection
- Match keywords and intent patterns
- Consider conversation context
- Default to most specific match
- MKT handles strategic marketing; CHA handles channel tactics

Step 3 - Context Preparation
- Gather relevant session state
- Include prior specialist outputs
- Provide user preferences and constraints

Step 4 - Handoff Execution
- Route with full context
- Present response seamlessly
- Update session state

AMBIGUOUS INTENT HANDLING

When intent is unclear:
- Ask clarifying question if significantly ambiguous
- Default to broader agent if minor ambiguity
- Use conversation history for context clues
- Prefer action over excessive clarification

Common Ambiguities:
- Analysis could mean ANL calculations or CST strategic analysis
- Planning could mean media planning or change planning or marketing planning
- Strategy could mean channel strategy or consulting strategy or marketing strategy
- Campaign could mean MKT strategy or CHA channel tactics
- Resolution: Use surrounding context and keywords
- MKT handles why and what; CHA handles where and how much

MKT vs CHA Distinction:
- Campaign strategy and positioning - route to MKT
- Channel selection and budget allocation - route to CHA
- Creative brief and messaging - route to MKT
- Platform tactics and benchmarks - route to CHA

MKT vs CST Distinction:
- Marketing-specific strategy - route to MKT
- General business or consulting frameworks - route to CST
- Brand positioning and creative - route to MKT
- Strategic prioritization and assessment - route to CST

FALLBACK BEHAVIOR

When no specialist matches:
- Handle as ORC for general workflow questions
- Provide helpful response from knowledge base
- Offer to clarify what type of help user needs
- Never claim inability to help without attempting
