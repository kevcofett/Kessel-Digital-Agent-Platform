OVERVIEW

This document provides agents with reference information for handling access-controlled content within MCMAP. The platform implements Attribute-Based Access Control (ABAC) integrated with the Mastercard Microsoft Directory.


PROTECTED CONTENT CATALOG

EXECUTIVE BRIEFS (PROTECTED)

- 00-MCMAP_CEO_Brief.md requires Executive Only access with employee_level IN CEO President EVP SVP OR security_group equals MCMAP-Executive-Access

- 00-A-MCMAP_Chief_Consulting_Officer.md requires Executive Only access with same requirements as CEO Brief

- 00-B-MCMAP_Chief_Technology_Officer.md requires Executive Only access with same requirements as CEO Brief

- 00-C-MCMAP_Chief_AI_Officer.md requires Executive Only access with same requirements as CEO Brief

- 00-D-MCMAP_Chief_Revenue_Officer.md requires Executive Only access with same requirements as CEO Brief

- 09A-MCMAP_Investment_Proposal.md requires Executive or Finance access with employee_level IN CEO President EVP SVP OR department IN Finance Strategy


PUBLIC CONTENT (NO RESTRICTION)

- 00-MCMAP_Glossary.md accessible to all authenticated users
- 00-MCMAP_Index.md accessible to all authenticated users
- 01-MCMAP_Executive_Summary.md accessible to all authenticated users with potential additional restrictions when Full ABAC Mode enabled
- 04-MCMAP_Agent_Capabilities.md accessible to all authenticated users with potential additional restrictions when Full ABAC Mode enabled
- 07-MCMAP_Operational_Runbook.md accessible to all authenticated users
- 10-MCMAP_Contact_Reference.md accessible to all authenticated users


TWO-MODE OPERATION

LAUNCH MODE (DEFAULT)

ABAC_ENABLED is false meaning full ABAC rules are dormant
CSUITE_PROTECTION_ENABLED is true meaning executive content is protected

Protected in Launch Mode
- CEO Brief and C-Suite role briefs 00-A through 00-D
- Investment Proposal 09A

Not Protected in Launch Mode
- All other technical documentation
- Glossary and Index
- Operational documentation


FULL ABAC MODE

ABAC_ENABLED is true meaning all ABAC rules are active
CSUITE_PROTECTION_ENABLED is true meaning executive content is protected

Additional Protections When Full ABAC Enabled
- Department-specific content such as KB-Marketing and KB-Consulting documents
- Regional benchmark data such as mpa_benchmark.region files
- Division-specific capabilities
- Financial benchmark data


ACCESS DENIAL HANDLING

DENIAL MESSAGE TEMPLATE

When a user is denied access to protected content use this response pattern

This content requires specific access permissions based on your role in the organization.

To request access say request access and I will help submit your request to the Platform team. They typically respond within 2 business days.

In the meantime I can help you with alternative content relevant to your role or general platform documentation or contact information for questions.


CRITICAL RULES

EMAIL PROTECTION - NEVER reveal the Platform Owner email address in denial messages

CONDITION PRIVACY - NEVER disclose specific access conditions or rules to users

NO WORKAROUNDS - NEVER suggest ways to bypass access controls

NO HINTS - NEVER hint at what attributes would grant access

GRACEFUL MESSAGING - ALWAYS use friendly helpful language in denials


ACCESS REQUEST WORKFLOW

WHEN USER SAYS REQUEST ACCESS

Step 1 Collect Information
Ask What content would you like access to

Step 2 Collect Justification
Ask Please briefly describe why you need this access in 1-2 sentences

Step 3 Confirm Submission
Say Your request has been submitted. The Platform team typically responds within 2 business days. Request ID is the ID from the flow response. Ask Is there anything else I can help you with in the meantime


INFORMATION CAPTURED AUTOMATICALLY

The access request flow automatically captures from the user directory profile
- Display name and email
- Job title and employee level
- Department team and division
- Region and country
- Manager chain

This information is included in the request email to the Platform Owner for decision-making.


AGENT INTEGRATION

DOCS AGENT RESPONSIBILITIES

When DOCS agent receives a request for protected content

1 Check Access by calling MCMAP_Check_Content_Access flow
2 If Granted return the requested content normally
3 If Denied display graceful denial message and offer alternatives
4 Offer Access Request by always mentioning the access request option


ORC AGENT RESPONSIBILITIES

When ORC routes to DOCS for executive content

1 Route Normally and let DOCS handle access verification
2 Do Not Pre-Check because access verification happens at DOCS level
3 Handle Returns normally when user returns from DOCS and continue session


USER PROFILE ATTRIBUTES

ATTRIBUTES AVAILABLE FOR ACCESS RULES

employee_level from Microsoft Directory with example values CEO EVP SVP VP Director Manager Analyst

department from Microsoft Directory with example values Marketing Services Finance Engineering

team from Extension attribute with example values Consulting Team A Analytics Team

division from Microsoft Directory with example values Data and Services Network Services

region from Usage location with example values North America Europe APMEA

country from Microsoft Directory with example values United States United Kingdom

security_groups from Azure AD with example values MCMAP-Executive-Access MCMAP-Leadership-Access

manager_chain from Microsoft Graph as Array of manager IDs up to CEO


CONDITION OPERATORS

SUPPORTED OPERATORS FOR ACCESS RULES

EQUALS for exact match example department EQUALS Finance

NOT_EQUALS for not equal example region NOT_EQUALS APAC

IN for value in list example employee_level IN EVP SVP VP

NOT_IN for value not in list example division NOT_IN Corporate

CONTAINS for string contains example job_title CONTAINS Director

STARTS_WITH for string starts with example department STARTS_WITH Marketing

MEMBER_OF for security group membership example security_group MEMBER_OF MCMAP-*

MANAGER_CHAIN for user in manager hierarchy example manager_chain CONTAINS user-guid


QUICK REFERENCE

CHECK IF CONTENT IS PROTECTED

1 Is it a CEO Brief or C-Suite brief 00-A through 00-D then Protected
2 Is it the Investment Proposal 09A then Protected
3 Is Full ABAC Mode enabled then Check mcmap_access_rules.yaml for additional rules
4 Otherwise Not Protected


HANDLE ACCESS DENIAL

1 Display graceful message and never mention specific rules
2 Offer alternative content
3 Offer access request option
4 Never reveal email addresses


PROCESS ACCESS REQUEST

1 Ask what content they want
2 Ask for brief justification
3 Submit via MCMAP_Access_Request flow
4 Confirm with request ID
5 Mention 2-day response SLA
